API du WebService

Cliquez sur les boutons "GET", "POST" et "PUT" pour voir les détails de chaque requête.
Attention: pensez bien à cliquer sur les ongles "Request" et "Response" ; il arrive que ceux-ci ne s'ouvrent pas automatiquement.

/prets

/prets get post

get

La liste de tous les prêts ayant un statut "prete" ou "rendu". Les prêts avec un statut "annule" sont ignorés. (Attention: charge au Javascript de "filtrer" les rendus des prêtés).

post

Création d'un prêt.

On remarquera que les valeurs possibles pour lestatutsont :

  • prete
  • rendu
  • annule

get /prets

La liste de tous les prêts ayant un statut "prete" ou "rendu". Les prêts avec un statut "annule" sont ignorés. (Attention: charge au Javascript de "filtrer" les rendus des prêtés).

  • Response

HTTP status code 200

La liste de tous les prêts.

Body

Type: application/json

Example:

[
  {"id": 169, "quoi": "Tablette iPad", "qui": "vdebaty", "statut": "rendu"},
  {"id": 170, "quoi": "Tablette ASUS", "qui": "bleroux", "statut": "prete"},
  {"id": 171, "quoi": "C\\u00e2ble DVI", "qui": "ppape", "statut": "prete"},
  {"id": 172, "quoi": "Macbook Air", "qui": "sguitoune", "statut": "rendu"}
]

post /prets

Création d'un prêt.

On remarquera que les valeurs possibles pour lestatutsont :

  • prete
  • rendu
  • annule

  • Request
  • Response

Body

Type: application/json

Example:

{"quoi": "C\\u00e2ble DVI", "qui": "ppape", "statut": "rendu"}

HTTP status code 201

Le prêt a bien été créé; on retourne l'URL du nouveau prêt.

Headers

  • location: required (string)

    L'URL du nouveau prêt.

    Example:

    /prets/173

Body

Type: text/plain

Example:

/prets/173

HTTP status code 400

Le prêt n'a pas pu être créé.
